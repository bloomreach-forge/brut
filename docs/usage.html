Usage Overview

BRUT provides three test types for brXM delivery tier testing. Choose based on what you're testing:

Testing... Use Guide
JAX-RS REST endpoints @BrxmJaxrsTest JAX-RS Testing
Page Model API responses @BrxmPageModelTest Page Model Testing
HST Components @BrxmComponentTest Component Testing

Quick Example


@BrxmJaxrsTest(resources = {HelloResource.class})
class MyApiTest {

    @Test
    void testEndpoint(DynamicJaxrsTest brxm) {
        String response = brxm.request()
            .get("/site/api/hello/world")
            .execute();

        assertEquals("Hello, World!", response);
    }
}

Key Features

  • Parameter Injection - No IDE warnings, standard JUnit 5 pattern
  • Zero-Config - Auto-detection of bean packages, HST root, Spring configs
  • Fluent Request API - brxm.request().get("/api").execute()
  • Typed JSON Responses - executeAs(User.class), executeWithStatus()
  • No Inheritance - Uses JUnit 5 extensions, not abstract base classes

Importing Nodes

You can import nodes using the ImporterUtils from the brut-common module.


import java.net.URL;
import org.bloomreach.forge.brut.common.repository.utils.ImporterUtils;

URL resource = getClass().getResource("/news.yaml");
ImporterUtils.importYaml(resource, rootNode, "/content/documents/mychannel", "hippostd:folder");

See Stubbing Test Data for YAML structure and patterns.
